# Bite-Log 設計ドキュメント

**作成日**: 2026-06-04  
**ステータス**: 承認済み  
**対象**: iPhoneで使うダイエット食事管理PWAアプリ

---

## 1. 概要

### 目的

Google Form + スプレッドシート + Looker Studio の現行システムを置き換え、iPhoneで手軽・高速に食事記録できるPWAアプリを構築する。写真またはテキストで食事を記録し、AIがカロリー・三大栄養素（PFC）を計算。モダンなダッシュボードでモチベーションを維持しながらダイエットを継続できる。

### 現行の課題と解決策

| 課題 | 解決策 |
|---|---|
| Google Formの入力が遅く画像アップロードが重い | PWAのネイティブカメラAPIで直接撮影・高速アップロード |
| 食事データをGoogle Healthアプリで確認したい | Google Health API v4にnutritionLogを書き込み。Google Healthアプリ（iOS）で食事・フィットネス・睡眠を統合表示 |
| ダッシュボードが1週間分しか見えずモチベーションがわかない | 週間・30日・開始〜今日の期間切替グラフ + 継続日数・達成率の可視化 |
| Geminiの計算がたまに失敗する | 同期型で結果表示、エラー時は再送信ボタンで対処。課金で安定化も可 |
| 適切なカロリー目標の設定方法がわからない | AIチャット + InBody画像OCRで基礎代謝実測値から高精度な目標を算出 |
| 週次フィードバックがなくモチベーションが続かない | AI週次・期間サマリー、連続記録ストリーク、達成率、InBodyスコア推移 |

### ユーザー

- **主要ユーザー**: 自分1人（個人ツール）
- **将来の拡張**: 他者が使えるUXを意識した設計（マルチユーザー対応は後フェーズ）

## 2. 技術構成

### フロントエンド

- **プラットフォーム**: PWA（Progressive Web App）
  - App Store審査不要・即時更新・iPhoneホーム画面追加でネイティブ感
  - iOS 16.4以降でプッシュ通知対応
- **フレームワーク**: React
- **スタイリング**: Tailwind CSS + shadcn/ui
- **ホスティング**: Vercel または Cloudflare Pages（無料枠）

### バックエンド / データ

- **データベース**: Google スプレッドシート（既存継続）
  - 3〜5食/日 ≈ 数MB/年。容量は実質無制限
  - 既存データ・Looker Studioとの互換性を維持
- **画像ストレージ**: Google Drive（既存継続）
  - 食事写真はDriveに保存し、URLのみスプレッドシートに記録

### AI

| 用途 | モデル | コスト |
|---|---|---|
| カロリー・PFC計算（食事写真/テキスト） | Gemini 2.0 Flash | 無料枠（1日1,500リクエスト）|
| InBody画像OCR・目標設定チャット | Claude API（Vision） | 〜¥30/月 |

### Google Health連携

- **対象アプリ**: Google Healthアプリ（iOS対応、2026年5月〜。旧Fitbitアプリ）
- **API**: Google Health API v4（`nutritionLog` データ型を使用）
- **認証**: 既存のGoogle OAuth 2.0に `health` スコープを追加
- **書き込みタイミング**: 食事記録時にGoogle Sheetsへの保存と同時に実行
- **書き込み内容**: 食事名（AIが認識した内容）・カロリー・タンパク質・脂質・炭水化物・食事タイプ・記録日時
- **コスト**: ¥0
- **注意**: 個人利用のPWAのためAPIアクセス審査は不要の可能性が高いが、実装時に確認必要

### 認証

- **Google OAuth 2.0**（「Googleでログイン」ボタン）
- 自分のGmailアカウントのみアクセス許可
- Google Sheets・Drive連携と同一アカウントで認証

### 画面5：AI目標設定チャット（設定から遷移）

会話形式で以下の情報を収集し、目標を自動算出・設定する。

**収集情報（通常モード）**:
- 現在の体重・身長
- 目標（体重-◎kg / 体脂肪率◎%以下など）
- 運動頻度・活動レベル
- 職業・日常活動量

**InBodyモード**:
- InBody結果画像をアップロード
- Claude Vision APIで自動OCR：体重・体脂肪量・筋肉量・体脂肪率・基礎代謝（実測値）・InBody点数を抽出
- 実測基礎代謝を使用した高精度な目標算出

**算出ロジック**:
```
目標カロリー = 基礎代謝 × 活動係数 − カロリー赤字幅
タンパク質   = 筋肉量 × 2.4g（筋肉維持優先）または体重 × 1.3g
脂質        = 目標カロリーの 20%
炭水化物     = 残りカロリー / 4
```

## 4. 食事記録フロー

```
アプリ開く
  → ＋ボタン（FAB）タップ
  → 記録画面
      ├─ 写真撮影（カメラ直接起動）または画像選択
      ├─ 食事内容メモ（テキスト）
      ├─ 食事タイプ選択（朝食/昼食/夕食/間食 ※時間帯から自動判定）
      └─ 備考（店名・メニュー名など。AI計算に優先反映）
  → 送信ボタン
  → [同期型] Gemini APIへ送信 → 2〜3秒待機
  → カロリー・PFC結果を表示
  → Google Sheetsに記録保存
  → 完了
```

**エラー時**: エラーメッセージ + 「再送信」ボタン表示。入力内容は保持。

## 6. 非機能要件

### パフォーマンス
- 記録フォームの初期表示: 2秒以内
- AI計算のタイムアウト: 10秒（超過時はエラー表示）
- ダッシュボードのデータ取得: 3秒以内

### セキュリティ
- Google OAuth 2.0認証必須
- 自分のGmailアカウントのみアクセス許可（allowlistで制御）
- HTTPS必須（Vercel/Cloudflare Pagesで自動対応）

### PWA要件
- ホーム画面追加対応（manifest.json）
- カメラAPI対応
- 基本的なオフラインキャッシュ（フォーム画面）

## 7. 実装フェーズ

スコープが広いため、以下の順番で段階的に実装する。

- Google OAuth認証
